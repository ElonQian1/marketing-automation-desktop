# Round 4｜执行策略（讲义总览）

> 目标（30–45 分钟）：统一“如何执行一条任务”的策略与口径：**API 优先 → 半自动兜底 → 话术与敏感词护栏**，确保稳、快、可追溯。

---

## 1) 本轮范围
- **执行顺序**：API 优先、权限不足或未开放 → 半自动兜底；
- **执前检查**：权限 / 频控 / 查重 / 敏感词；
- **话术与变量**：统一模板、变量规范与多样性策略；
- **审计与回证**：每次执行可被复盘；
- **运营操作台**：执行态的 UI 反馈与可解释性。

---

## 2) 执行决策矩阵（简版）
| 任务类型 | 来源 | 是否开放 API | 我们的动作 |
|---|---|---|---|
| 回复评论 | 抖音（自然） | 已获权限 | **走 API**（执前四件套→调用→签收） |
| 回复评论 | 抖音（自然） | 权限不足 | **半自动**（跳转定位→预填话术→人工发送→签收） |
| 回复/隐藏 | 巨量（广告） | 是 | **走 API**（广告评论管理） |
| 关注 | 任意 | 视平台而定 | 无开放则**半自动**，有则**API** |

> 统一口径：**能 API 就 API**；否则**半自动**，但**执前四件套**（频控/查重/敏感词/权限）一律要过。

---

## 3) 执前四件套（必做）
1. **权限**：是否具备此动作的 scope；令牌是否有效；
2. **频控**：账号配额/间隔、视频/用户窗口限制；
3. **查重**：评论级/用户级/文本级；
4. **敏感词**：命中 block 则禁止；review 则需人工确认或改写。

---

## 4) 执行闭环（通用）
1) **预览**：显示评论上下文、视频预览、历史触达；
2) **话术**：从模板生成可编辑草稿（变量自动填充、可多条候选）；
3) **执行**：API 或半自动；
4) **签收**：写 `audit_logs`，更新 `tasks`，必要时回填“已发送文本”；
5) **回证**：若 API 返回 message_id/URL 则落库；半自动由运营勾选“已发送”并记录方式。

---

## 5) 风险最小化
- 默认开启“**随机化**”与“**多模板**”；
- 热点期增加间隔与模板多样性；
- 异常高失败率自动**降速/熔断**。

---

## 6) 本轮输出物
- 《API 执行器与鉴权（v1）》
- 《半自动兜底流程（v1）》
- 《话术模板与变量规范（v1）》
- 《敏感词策略与内容多样性（v1）》
- 《执行台操作手册（v1）》
- 《演练脚本（执行策略）》

---

## 7) 验收（DoD）
- 任一任务进入执行页时，执前四件套均可见且可解释；
- API 与半自动两条路径都能完成签收与审计；
- 敏感词命中与文本相似度超阈值时有明确提示与替换路径；
- 运营能在 3 步内完成“选择话术 → 发送 → 签收”。
